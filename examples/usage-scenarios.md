## 示例 1：公司研究（深度版）

> **研究对象：** 字节跳动

### 研究框架

**📈 纵向分析**
```
起源：今日头条 (2012) → 算法推荐驱动
关键节点 1：2016 年上线抖音 → 从信息分发给短视频的战略延伸
关键节点 2：2020 年 TikTok 全球爆发 → 从国内到国际的跨越
关键节点 3：2021 年组织架构调整 → 从"App工厂"到大模型布局

决策逻辑分析：
- 为什么今日头条成功了？→ 推荐算法的先发优势和持续迭代
- 为什么做抖音？→ 看到短视频趋势 + 头条的技术复用
- 为什么 TikTok 能出海？→ 买 Musical.ly 获取海外根基
- 为什么调整组织？→ 应对监管 + 多业务线管理需要

路径依赖：
- 算法基因 → 所有产品都有推荐系统
- App 工厂模式 → 快速试错、批量孵化
- 数据驱动文化 → 优势（效率高）和短板（ToB 产品缺少同理心）
```

**📊 横向分析**
```
竞品对比：

| 维度 | 字节跳动 | 腾讯 | Meta |
|------|---------|------|------|
| 核心能力 | 算法推荐 + 内容生态 | 社交关系链 | 社交图谱 |
| 产品矩阵 | 抖音/TikTok + 今日头条 | 微信/QQ + 视频号 | FB/IG/WhatsApp |
| AI 布局 | 豆包 + 火山引擎 | 混元大模型 | Llama 系列 |
| 国际化 | TikTok 全球领先 | 微信出海遇阻 | 成熟全球网络 |
| 收入模式 | 广告 + 电商 + 直播 | 游戏 + 广告 + 金融 | 广告 + VR |

生态位：全球唯一的"算法+内容+国际化"三位一体选手
互补关系：火山引擎与云厂商竞争；豆包与大模型赛道竞争
```

**🎯 横纵交汇**
```
关键洞察：
1. 字节的算法优势不是新技术，而是从 2012 年持续迭代 10+ 年的积累
2. TikTok 的成功 = 今日头条的算法基因 × Musical.ly 的海外根基
3. 组织调整（2021）和 AI 转型（2023+）是路径依赖的自然延续

未来 1-3 年判断：
- 国际化仍是核心增长引擎（但地缘风险加剧）
- AI 大模型布局决定下一个十年的位置
- 关键变量：TikTok 的命运 + 豆包的市场占有率
```

## 示例 2：技术概念研究（快速版 3 分钟）

> **研究对象：** WebAssembly (Wasm)

### 快速分析

**📈 纵向**
```
起源：2015 年由 Mozilla/Google/Microsoft/Apple 联合发起
转折 1：2017 年 MVP 落地 → 浏览器端高性能计算
转折 2：2019 年 WASI 发布 → 从浏览器走向服务端
转折 3：2023 年 Component Model 推进 → 跨语言组件生态
决策逻辑：从一开始就设计为"编译目标"而非"手写语言"，绕开了 JS 性能瓶颈
```

**📊 横向**
```
竞品：
- JS/Node.js：生态最大，但性能上限明显
- Docker：容器化代表，与 Wasm 非直接竞争而是互补
- Python：AI 生态王者，在边缘计算场景与 Wasm 竞争
- Flutter/Dart：跨平台方案，但需自带 Runtime

优势：冷启动快（ms 级 vs Docker 秒级），安全沙箱，多语言支持
短板：生态不成熟，调试困难
```

**🎯 交汇**
```
Wasm 的路线（从浏览器到通用 Runtime）和 Docker 的历史路径（从容器化到编排平台）非常相似。
关键变量：WASI 标准化进度 + 云厂商支持力度。
未来 1-3 年：边缘计算 + Serverless 场景最先落地，替代 Docker 在轻量场景的位置。
```

## 示例 3：个人选择（横纵分析自己）

> **研究对象：** 个人职业发展

### 快速分析框架

```
📈 纵向（自我复盘）
- 起源：当初为什么选择这个方向？
- 转折点：做过哪些关键决定？当时的考量是什么？
- 能力积累：什么能力是持续在用且在成长的？
- 瓶颈：目前在什么阶段卡住了？

📊 横向（定位对比）
- 同龄人对比：同专业/同行业的同龄人在做什么？
- 岗位对比：目标岗位需要的技能栈 vs 自己已有的
- 趋势对比：行业正在朝哪个方向走？自己的方向对不对？

🎯 交汇
- 过去的积累路径和现在市场需要什么有什么交叠？
- 判断下一步应该深耕还是换道？
```

## 💡 使用技巧

| 技巧 | 说明 |
|------|------|
| 🎯 **选对版本** | 深度版出报告，快速版出判断，别搞混 |
| 🔄 **填充具体** | 把 `[研究对象]` 替换为具体名称，不要用模糊代称 |
| 📝 **引导 AI** | 在 prompt 中指明你想要的重点维度 |
| 🔗 **交叉验证** | 关键信息（如公司数据）建议人工核实 |
| 🧩 **多对象对比** | 同时分析 2~3 个对象，得到更全面的生态位图景 |
